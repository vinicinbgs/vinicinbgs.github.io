# Requirements

### Functional
- What the system should do.

### Non-Functional
- Functionality
- Reliability
- Usability
- Efficiency

Examples:
- Developed for several years (maintainability)
- Able to serve millions of user (scalability)
- Available 24/7 (Reliability)
- Short latency response (Efficiency)
